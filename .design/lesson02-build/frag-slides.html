一条 print()，输出一行
main.py — python3
print("第一行")
print("第二行")
print("第三行")
$ python main.py
第一行
第二行
第三行
Python 按代码顺序，从上到下执行。
一次输出多个内容
print("答案是", 42)
print("不要慌")
多个内容用逗号隔开；输出时默认用空格分隔。
先猜：会输出几行？
print("当前回合", 3)
print("得分", 10, "分")
揭示：共 2 行；每行参数之间自动出现空格。
点击揭示
# 后面，是写给人看的说明
# 下面一行显示程序标题。
print("今天也要写出能运行的代码")
# TODO：以后再让程序自动写作业。
注释不会被 Python 执行，也不会出现在输出中。
好注释，解释“为什么”
# 分开输出，方便定位故障。
print("模块 A 正常")
print("模块 B 正常")
不必把每行代码再翻译一遍。
NEW WORDS · 三个新词
出错，不等于失败
Bug：程序里的错误
Debug：定位、修复、再验证
Traceback：错误追踪信息
语法结构不完整
main.py — python3
print("少了右括号"​
$ python main.py
SyntaxError: '(' was never closed
先检查成对符号：括号、引号。
Python 不认识这个名称
main.py — python3
pritn("拼写也很重要")
$ python main.py
NameError: name 'pritn' is not defined
对照正确名称，逐字符检查拼写。
不该缩进的地方，多了空格
main.py — python3
print("意外缩进")
$ python main.py
IndentationError: unexpected indent
Python 会认真对待每一层缩进。
语法正确，运行时仍可能出错
main.py — python3
print(10 / 0)
$ python main.py
ZeroDivisionError: division by zero
程序开始执行后，遇到了不允许的运算。
错误类型与行号，都在里面
Traceback (most recent call last):
File "main.py", line 1, in <module>
pritn("拼写也很重要")
NameError: name 'pritn' is not defined
从最后一行找类型；再向上找 line 1。
①最后一行：错误类型
②向上找：行号 line 1
③回到代码：pritn
Debug 不是乱改代码
停下来 → 读最后一行 → 找类型与行号
检查代码 → 猜一个原因 → 只改一处 → 再运行
一次只验证一个猜想。
修好一个，才会看到下一个
print("第一处少了括号"
pritn("第二处拼错了")
Python 通常先报告最早挡住执行的问题。
修复后必须重新运行。
断点：让程序暂停给你看
● print("第 1 步：准备")
○ print("第 2 步：执行")
○ print("第 3 步：完成")
断点不是错误；它是你主动放下的暂停标记。
暂停以后，怎么走？
1.行号左侧点出红点
2.使用 Debug 启动
3.Step Over：执行当前行
4.Resume：继续运行
