# 火星基地资源调度模拟器：教师参考实现

本目录保存贯穿项目的最终参考版本，用于验证项目规则、控制最终代码规模，并作为后续拆分 V0-V8 章节版本的基线。

`mars_base_simulator.py` 是教师参考答案，不建议在课程开始前直接发给学生。

逐章递进代码见 [V0-V8 章节版本](../chapter-versions/README.md)。

## 运行程序

在项目根目录执行：

**Windows**

```powershell
py project/reference-solution/mars_base_simulator.py
```

**macOS / Linux**

```bash
python3 project/reference-solution/mars_base_simulator.py
```

## 教学语法边界

参考程序刻意保持为单文件顺序代码，并且不使用：

- 自定义函数和类
- 列表、元组、字典和集合
- 文件读写
- 异常处理
- 导入和第三方库

这些限制并不是生产项目的代码组织建议，而是为了保证学生能够只使用当前阶段已经学习的内容理解完整程序。

参考程序实际覆盖：

- 注释和 `print()`
- 变量、数值、字符串和布尔值
- f-string 和字符串重复
- `input()`
- 算术、赋值、比较和逻辑运算
- `if`、`elif`、`else` 和嵌套判断
- `while`、`for`、`range()`
- `break` 和 `continue`

## 核心规则

模拟器持续 7 个有效任务日。每天选择一项任务：

| 编号 | 决策 | 直接效果 |
| --- | --- | --- |
| `1` | 维护供能系统 | 能源 `+25` |
| `2` | 维护生命保障系统 | 能源 `-15`，氧气 `+20`，饮用水 `+15` |
| `3` | 开展科研探索 | 能源 `-20`，氧气 `-10`，饮用水 `-8`，科研积分 `+30` |

每个有效任务日结束后，再扣除基础消耗：能源 `-5`、氧气 `-10`、饮用水 `-8`。资源最大值为 `100`。

输入无效编号或资源不足以开展科研时，不消耗任务日。任一关键资源小于等于 `0` 时，任务进入应急撤离流程。

## 代表性测试序列

输入基地名称后，依次输入下面的任务编号：

| 决策序列 | 预期结果 |
| --- | --- |
| `1 1 1 1 1 1 1` | 基地稳定运行，科研目标未完全达成 |
| `3 3 1 1 1 1 1` | 火星任务顺利完成 |
| `3 3 3 2 1 1 1` | 卓越科研任务 |
| `3 3 3 3` | 第 4 个任务日能源耗尽，应急撤离 |

## 运行自动测试

测试从程序外模拟键盘输入，不要求修改学生参考代码。

```powershell
py -m pytest tests/test_mars_base_simulator.py -q
```

测试覆盖四类任务结果、无效选择、默认基地名，以及禁止超纲语法的课程约束。
