# 双项目最终参考实现

本目录对应 [`双项目贯穿式实践实施计划`](../../docs/贯穿式项目实施计划.md)，保留两个互相独立的教师参考与恢复版本。它们展示预期行为和可测试结构，不是要求学生逐行照抄的唯一答案。

## 项目一：我的一周预算助手

运行：

```powershell
python project/reference_solution/budget_assistant.py
```

程序接收正整数预算和多笔支出，支持餐饮、交通、学习、其他四类记录。每次记录后显示剩余金额、已使用比例和预算状态，输入 `Q` 后生成分类汇总、平均每笔支出与建议。

默认以预算的 `80%` 作为提醒线，并单独处理“恰好用完”和“已经超支”。提醒线只是用户工具的默认值，不代表统一理财标准。核心版本只保存本次运行的数据，文件存档属于课后扩展。

代表性测试：正常使用、达到 80%、恰好用完、超支、无效金额、无效类别和零笔记录。

## 项目二：群星彼岸·候选行星分析器

运行：

```powershell
python project/reference_solution/planet_analyzer.py
```

程序先按基准规则分析四颗候选。输入编号可查看证据，输入 `P` 可在深拷贝上预演更保守的规则，输入 `Q` 生成最终科研报告。

基准规则：

- 温度 `-20–40 °C`
- 重力 `0.65–1.30 g`
- 大气压力 `0.50–1.50 atm`
- 辐射等级不高于 `5`
- 所有区间上下边界均算通过

超出硬指标时为“不建议继续”；没有硬风险但缺少水信号或关键观测时为“需要更多观测”；其余为“优先候选”。这些规则是当前课程采用的教学化模型，不代表真实宜居性结论。

`preview_rule_changes()` 会复制规则、候选、风险标签和报告组成的完整嵌套状态。预演对副本的修改不会污染基准研究。

代表性测试：三类结论、闭区间边界、硬风险优先级、批量报告、深拷贝隔离、规则变化比较和完整交互。
