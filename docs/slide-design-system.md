# 课件幻灯片设计系统（三风格体系）

> 版本：1.0
> 日期：2026-07-19
> 适用范围：`lessons/*/slides.html` 全套章节课件
> 技术底座：项目技能 `frontend-slides-editable`（单文件 HTML + 浏览器内可编辑运行时）

## 1. 设计原则

1. **只上学生要看的东西**：幻灯片是给学生看的投影，不是教案的搬运。教学目标、验收标准、教师口令、助教分工、进度调整一律留在 `teacher-guide.md`，绝不上屏。每页内容先问一句：学生抄不下、看不懂、用不上的，删掉。
2. **动效服务节奏**：课堂大量时间是学生动手，幻灯片是"讲"的骨架。动画只用于关键揭示时刻，翻页必须即时（键盘 ←/→/Space）。
3. **一页一个知识点**：对应课程设计"一次只增加一个主要难点"，溢出就拆页，页内永不滚动。
4. **先预测，再运行**：预测题统一做成"先表态、点击揭示"的交互页，配合课堂举手/绿黄红卡。
5. **三风格即三种教学信号**：学生看到画面风格就知道接下来要干什么——听讲、看案例，还是进入新环节。
6. **Python 是主线，火星是案例**：火星基地只在"贯穿项目"环节出现（风格 C 的页面）。其余页面一律以 Python 知识本身为主角，不出现火星元素、基地术语和资源数值——避免学生每页都要重新解析同一套情境。

## 2. 三风格分工

| 风格 | 代号 | 承载内容 | 视觉基调 |
| --- | --- | --- | --- |
| 瑞士纸白 | **S** | 封面、背景导入、章节地图、环节切换宣言、检查点、总结 | 纸白 + 墨黑 + 安全橙，信息设计，文化感 |
| 终端绿 | **B** | 概念讲解、语法拆解、代码逐行执行、Traceback 阅读、故障速查 | GitHub 暗色 + 终端绿，CRT 扫描线，打字机动画 |
| 像素轨道 | **C** | 贯穿项目版本（V0–V8）、玩梗输出、独立任务、预测互动 | 深空 + 像素街机，霓虹粉/青/黄，硬阴影 |

典型一章的节奏：`S 开场 → B 讲概念 → C 项目案例 → B 排错 → S 检查点与总结`。

## 3. 风格 S · 瑞士纸白（叙事页）

```css
--s-bg: #FAFAF8;      /* 纸白 */
--s-ink: #0A0A0A;     /* 墨黑 */
--s-accent: #FF6B35;  /* 安全橙 */
--s-muted: #6B6B66;
--s-hairline: rgba(10,10,10,.14);
```

- **字体**：拉丁用 `Archivo`（800 标题 / 400-500 正文）；中文标题用宋体类（`"Songti SC","STSong","SimSun","Noto Serif CJK SC",serif`）制造文化感，正文用黑体类原生 CJK 栈。
- **签名元素**：可见细网格/十字对位标记、超大章节数字（描边或浅灰）、不对称编辑式排版、橙色粗规矩形或水平粗线、全大写加宽字距的 eyebrow 小标、发丝线分隔。
- **动效**：克制的 staggered fade-up；无粒子、无扫描线。
- **切换宣言页**：整页一句话超大字，是环节转换的"锣声"。

## 4. 风格 B · 终端绿（知识页）

```css
--b-bg: #0d1117;
--b-panel: #161b22;
--b-border: #30363d;
--b-green: #39d353;
--b-text: #c9d1d9;
--b-muted: #8b949e;
--b-red: #ff7b72;   /* 仅用于报错 */
```

- **字体**：`JetBrains Mono` + 原生 CJK 栈；全页等宽。
- **签名元素**：CRT 扫描线覆盖层、终端窗口卡（三色灯标题栏 + 路径）、打字机逐字动画（`steps()`）、闪烁方块光标、语法着色、`$` 提示符。
- **代码页规范**：左代码右输出，或上代码下要点 chips；代码 ≤10 行；Traceback 用 `--b-red`，逐行高亮阅读顺序。
- **动效**：打字、行级点亮（`animation-delay` 逐行）、错误页轻微震动一次后静止。

## 5. 风格 C · 像素轨道（案例页）

```css
--c-void: #070a1a;
--c-pink: #ff2e88;
--c-cyan: #3ee6ff;
--c-yellow: #ffd23f;
--c-text: #eef1ff;
--c-muted: #9aa3c7;
```

- **字体**：标题 `Tektur`（800），正文/代码 `Chakra Petch` + 原生 CJK 栈。
- **签名元素**：`box-shadow` 像素星野（闪烁）、CRT 扫描线、像素角括号面板、硬偏移阴影（`6px 6px 0`）、虚线轨道环 + 卫星方块、像素火星（粉色圆角方块 + 光晕）。
- **项目页规范**：版本号徽章（V0…V8）+ 代码 + 输出面板并排；任务要求用黄色"像素按钮"芯片。
- **动效**：`popIn` 弹性进入、星星闪烁、轨道慢旋转；标题硬阴影 `4px 4px 0 var(--c-pink)`。

## 6. 交互约定

| 交互 | 用法 | 实现 |
| --- | --- | --- |
| 页内分步揭示 | 仅限**结果性质**（预测后的输出/报错/答案）与**步骤性质**（需要逐段讲解的拆解）元素 | 被标记 `data-step` 的元素进入页面时保持隐藏；→/Space/点击 = 揭示下一组，全部揭示完才翻页；← = 收回一组；跳转 = 直达终态。其余元素的入场动画在翻页到位后自动播放，无需手动触发 |
| 点击揭示 | 预测题、三问互动 | `data-reveal` 元素，演示模式下点击切换 `.revealed`；编辑模式禁用 |
| 键盘翻页 | 全程 | 运行时自带 ←/→/Space（与分步揭示共用按键） |
| 章节跳转 | 教师跳过已讲模块 | Pages 侧栏缩略图 |
| 激光笔 | 投影指示 | 运行时自带（左上角悬停） |
| 课前改字 | 教师临时调整 | `E` 进编辑模式，Ctrl+S 保存；编辑模式下所有分步元素全部可见可编辑 |

**分步标记规范**：默认全部自动播放，只有以下两类元素加 `data-step` 标记转为手动触发——结果性质（预测之后才展示的输出、报错、答案）和步骤性质（需要逐段讲解的代码拆解、流程点亮）。每页手动步 ≤3 组，其余元素入场自动。分步状态、揭示状态均为瞬态：不进历史栈、不进 localStorage、不进导出文件；`prefers-reduced-motion` 下全部直接显示终态。

**第 1 章的手动触发清单**（后续章节照此粒度）：

| 页 | 手动内容 | 性质 |
| --- | --- | --- |
| P8 第一个程序 | 输出 `Hello, Python!`（打字机自动，输出手动） | 结果 |
| P9 看懂这一行 | 三段拆解逐段点亮 | 步骤 |
| P11 字符串与引号 | 运行结果区（data-reveal 点击） | 结果 |
| P12 先猜再运行 | 揭示行（data-reveal 点击） | 结果 |
| P13 故意弄坏它 | 报错块 | 结果 |
| P16 案例 V0 | 输出面板 | 结果 |
| P18 运行链路 | 流程三段逐段点亮 | 步骤 |
| P19 三问互动 | 三张答案卡（data-reveal 点击） | 结果 |

## 7. 技术与交付约定

- 每章一个文件：`lessons/NN-<slug>/slides.html`，单文件自包含。
- 拉丁字体（JetBrains Mono / Tektur / Chakra Petch / Archivo）优先内联 base64 `@font-face`（latin 子集）；失败则分层回退栈。**禁止**残留外链字体 `<link>`。
- CJK 一律原生系统栈，不引网络字体。
- 每页 `section.slide` 有稳定 `id`；可拖动内容在 `.slide-edit-layer` 内挂 `data-slide-object` + 唯一 `data-oid`。
- 每页 `height:100dvh; overflow:hidden`，字号间距全用 `clamp()`，1280×720 下无页内滚动。
- `data-mobile-adaptation="desktop-default"`（教室投影优先）。
- 嵌入完整可编辑运行时（Pages 侧栏 / 撤销重做 / 激光笔 / 全屏 / Ctrl+S / 导出 HTML）。
- 预览参考：`.design/slide-previews/style-b.html`、`style-c.html`（A 为未采用的备选，保留备查）。

## 8. 批量生产检查单

每章交付前过一遍：

- [ ] 每页内容都是"给学生看的"，无教学目标/验收/教师口令类文字
- [ ] 火星基地元素只出现在贯穿项目页，其余页以 Python 为主线
- [ ] 三风格使用符合分工表，没有混用签名元素
- [ ] 每页 ≤ 密度上限（代码页 ≤10 行，要点 ≤6 条）
- [ ] 预测题全部有点击揭示，且与教案"先预测再运行"对应
- [ ] 1280×720 无页内滚动；`prefers-reduced-motion` 可用
- [ ] 编辑模式：改字、拖动、Pages 调序、导出 HTML 均正常
- [ ] 教案 `teacher-guide.md` 已同步（幻灯片入口、课堂口令变化）
- [ ] 章节课件先产 `slides-content.md` 内容稿，教师审核修改后再生成 HTML

## 9. 投影可读性与布局规范（硬指标）

教室投影、学生远距离观看，按"最后一排能看清"设计：

**字号下限（1280×720 基准）**

| 元素 | 下限（clamp 的 min 值） | 说明 |
| --- | --- | --- |
| 页面主标题 | 40px（约 4.5vw 起） | 一页只有一个视觉主角 |
| 正文 / 要点 | 24px（约 2.2vw 起） | 低于此值的内容一律删或拆 |
| 代码 | 26px（约 2.4vw 起） | 代码是这门课的主角，宁大勿小 |
| 辅助小字 | 18px（约 1.6vw 起） | 每页至多一行，能不用就不用 |

**密度上限（比第 7 节更严）**

- 要点每页 ≤ 4 条，单条 ≤ 16 字；代码每页 ≤ 8 行
- 一页总字数（不含代码）≤ 80 字；超过就拆页

**布局聚拢**

- 主内容容器 `max-width ≤ 76vw`，水平垂直居中于屏幕；视觉重心在屏幕中轴附近
- 装饰元素（网格、星野、扫描线、幽灵数字）只作边缘低对比陪衬，不占据视觉通道
- 页内留白集中在容器外围，不在内容元素之间撑大空隙；元素间距用节奏化的固定档位，不用"均分撑满"
- 同一页最多 2 个视觉焦点（如 标题 + 代码卡）；四角不得同时出现独立信息块
