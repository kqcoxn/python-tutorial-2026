## 1. 实施目标

课程使用两个可以独立完成的小项目组织主要实践：

- **生活项目：我的一周预算助手**，在第 1–6 课完成，证明 Python 能直接处理日常生活中的记录、计算和决策问题。
- **科研项目：群星彼岸·候选行星分析器**，从第 7 课后 25 分钟开始，在第 12 课完成，训练函数、数据建模、方案隔离、证据解释和知识迁移。

两个项目不并行维护。预算助手完成后先保留可运行版本，再通过一次函数化重构建立迁移桥梁；行星分析器从新文件开始，不依赖预算项目内部变量。

## 3. 项目一：我的一周预算助手

### 3.1 用户问题

刚进入大学的学生需要在一周内控制餐饮、交通、学习用品和其他支出。程序帮助用户连续记录金额，及时看到剩余预算和风险状态，并在结束时生成汇总。

核心版本只管理当前运行中的数据，不写入文件。这里的“实用”来自真实输入、即时计算、明确反馈和可解释建议，不依赖复杂界面。

### 3.2 核心状态

在尚未系统学习容器前，使用清楚的独立变量：

```python
weekly_budget = 500
total_spent = 0
expense_count = 0
food_total = 0
transport_total = 0
study_total = 0
other_total = 0
```

第 7 课函数化重构可以拆分输入、状态判断和报告显示，但不强制把项目改写为复杂嵌套容器。

### 3.3 判断规则

| 状态 | 条件 | 输出重点 |
| --- | --- | --- |
| 正常 | 支出低于预算的 80% | 剩余金额与可继续使用比例 |
| 接近上限 | 支出达到或超过 80%，但未用完 | 提醒复核后续必要支出 |
| 恰好用完 | 支出等于预算 | 明确边界，不误判为超支 |
| 已超支 | 支出大于预算 | 显示超出金额并建议停止非必要支出 |

80% 是工具默认提醒线，不是普遍正确的理财标准。学生可以在扩展中修改提醒线并解释变化。

### 3.4 最终流程

```text
输入一周预算
-> 输入支出金额或结束指令
-> 选择消费类别
-> 更新总支出和分类小计
-> 显示剩余金额与预算状态
-> 继续记录或结束
-> 输出总额、分类小计、平均支出和建议
```

### 3.5 验收合同

- 程序可独立运行并由用户主动结束。
- 预算和支出必须为正整数；无效文本不会进入计算。
- 至少支持餐饮、交通、学习和其他四个固定类别。
- 循环中的金额、计数器和分类小计更新一致。
- 至少测试正常、恰好用完和超支三类场景。
- 学生能解释循环终止条件和一个边界判断。

## 4. 项目二：群星彼岸·候选行星分析器

### 4.1 科研问题

研究小组收到多个候选行星的简化观测记录，需要按本次任务采用的筛查规则逐项分析，保存风险标签，比较不同规则方案，并提交候选结论和证据限制。

程序是数据分析工具，不再模拟飞船航行。学生操作的是观测记录、规则和报告，而不是燃料、任务时间或随机事件。

### 4.2 最终状态模型

```python
research = {
    "rules": {
        "temperature_range": (-20.0, 40.0),
        "gravity_range": (0.65, 1.30),
        "pressure_range": (0.50, 1.50),
        "max_radiation": 5,
    },
    "planets": [
        {
            "name": "Aster-4b",
            "temperature_c": 18.0,
            "gravity_g": 0.92,
            "pressure_atm": 1.10,
            "water_signal": True,
            "radiation_level": 2,
            "observation_complete": True,
            "observation_note": "基础观测已完成",
            "risk_tags": set(),
        }
    ],
    "reports": [],
}
```

### 4.3 筛查规则

- **优先候选**：温度、重力、大气压力和辐射通过本次任务规则，存在液态水信号，且要求的观测完整。
- **需要更多观测**：没有硬风险，但缺少明确水信号或关键观测不完整。
- **不建议继续**：至少一项温度、重力、大气压力或辐射硬指标超出范围。

范围上下边界均算通过。报告必须列出理由，并声明这是教学化筛查，不代表真实宜居性结论。

### 4.4 候选数据角色

正常路线使用四颗候选：

1. 多项指标均衡的优先候选。
2. 有水信号但重力触发硬风险的反例。
3. 基础条件有潜力但关键观测不完整的待观测候选。
4. 多项指标恰好位于接受边界的边界测试候选。

数据包必须让三类结果全部可到达，并覆盖区间下边界和上边界。

### 4.5 最终流程

```text
读取候选目录和基准规则
-> 逐个调用同一分析函数
-> 收集结论、理由和唯一风险标签
-> 生成基准研究报告
-> 深拷贝研究状态
-> 在副本中应用保守规则并重新分析
-> 比较结论变化
-> 验证基准状态未被污染
```

### 4.6 函数职责

| 函数 | 输入 | 返回或副作用 |
| --- | --- | --- |
| 分析单个候选 | 行星字典、规则字典 | 结论、理由和风险标签 |
| 分析候选目录 | 研究状态 | 更新报告和星球风险标签 |
| 生成报告 | 研究状态 | 返回可显示文本 |
| 预演规则方案 | 研究状态、规则修改 | 返回独立的模拟状态 |
| 比较方案 | 两组报告 | 返回结论发生变化的候选 |

### 4.7 验收合同

- 至少三个候选经过同一个函数分析。
- 三类筛查结果均由测试数据到达。
- 列表表示候选序列，字典表示字段记录，集合表示唯一风险标签，元组表示固定区间。
- 深拷贝规则方案后，基准规则、报告和星球标签保持不变。
- 最终报告包含证据、风险或缺失信息以及限制声明。
- 至少测试正常候选、规则边界和共享状态污染三类情况。

## 6. 关键课堂活动

### 第 1–3 天：生活项目

- 用真实但不敏感的示例金额完成预算追踪。
- 先预测重新赋值和累计结果，再运行程序。
- 为 80% 提醒线和恰好用完设计边界表。
- 修复字符串与整数混用、计数器不更新和循环无法终止。
- 第 6 课保留 15 分钟完成独立使用、测试和说明。

### 第 4 天：迁移桥梁

- 先把预算状态判断拆成“计算”和“显示”函数。
- 比较硬编码变量与函数参数的差异。
- 给出单颗行星记录，要求学生用同样的输入—处理—输出思路设计分析函数。
- 明确指出迁移的是问题结构，不是变量名称。

### 第 5–6 天：科研项目

- 用列表和字典建立候选目录，避免平行列表。
- 使用集合收集去重后的风险标签，使用元组保存固定区间。
- 先画引用关系图，再比较赋值、浅拷贝与深拷贝。
- 最后一课使用未提前公布结论的数据完成盲测，不现场重写主流程。

## 7. 测试与成果证据

| 天次 | 必须保留的证据 |
| ---: | --- |
| 1 | 能独立运行和修改程序；一次变量状态追踪 |
| 2 | 单笔支出输入—计算—输出；预算边界表；一次报错修复 |
| 3 | 会终止的记录循环；预算助手；正常、用完、超支测试 |
| 4 | 函数契约；生活到科研的结构映射；一次返回值错误修复 |
| 5 | 候选数据模型；容器选择解释；批量分析版本 |
| 6 | 引用图；拷贝污染修复；最终科研报告和限制说明 |

正常路线和最低路线均提交两个独立成果；最低路线可以缩减字段、候选数量和交互包装，但不合并或取消其中一个项目。最终提交组合：

1. 一个可独立运行的预算助手。
2. 一个可独立运行的候选行星分析器。
3. 两个项目各至少三类代表性测试：预算侧为正常、恰好用完和超支；科研侧为正常候选、规则边界和共享状态污染。
4. 一项共享状态或深浅拷贝的预测、修复和解释。
5. 一段简短反思：同一问题解决方法怎样从生活问题迁移到科研问题。

## 8. 自由扩展

### 预算助手

- 自定义提醒比例或增加储蓄目标。
- 增加某一类别的独立上限。
- 比较计划支出与实际支出。
- 课后使用 CSV 或 JSON 保存记录。

### 行星分析器

- 增加观测置信度或恒星类型。
- 比较更宽松或更保守的规则。
- 增加缺失字段的处理策略。
- 使用简单排序生成后续观测优先序。

扩展必须先通过核心检查点，并说明新增需求、修改的状态或规则以及验证方法。

## 9. 进度调整

### 进度落后

- 预算助手仍保留四类固定支出，但可以减少每类示例和自由扩展；不能删除连续记录和边界判断。
- 行星候选从四个减为三个，但必须覆盖三类结论。
- 规则方案比较可以作为独立拷贝故障任务，不并入最终交互。
- 省略自由扩展和集中展示，保留独立编写、测试和解释。

### 进度超前

1. 增加无效输入、边界和反例。
2. 比较不同提醒线或筛查规则。
3. 改进函数接口并减少副作用。
4. 使用允许的扩展语法。
5. 课后再进入文件保存、真实数据或可视化。

## 10. 主要风险与控制

| 风险 | 控制方式 |
| --- | --- |
| 两个项目造成频繁切换 | 前三天完成生活项目，后三天完成科研项目，只在第 7 课做一次显式迁移 |
| 预算项目过于简单 | 要求真实循环、状态更新、边界测试和最终报告，不停留在一次减法 |
| 预算数据涉及隐私 | 默认使用虚构金额，不要求公开个人真实消费 |
| 科研项目仍像换皮游戏 | 移除飞船资源与事件，直接处理观测、规则和研究报告 |
| 科学规则被误认为定论 | 始终称为教学化筛查规则，并输出限制声明 |
| 累计错误阻断后半程 | 科研项目从新文件和可运行起点开始，不依赖预算文件内部状态 |
| 深拷贝被当作万能方案 | 先说明方案需要独立所有权，再选择深拷贝并验证基线不变 |
