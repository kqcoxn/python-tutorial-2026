# 课程 PPT 双风格参考

> 状态：四主题视觉规范（瑞士纸白 / 终端绿 / 账本蓝 / 星图青）；§6 记录 M0 样板阶段决策，§7–§8 定义双项目平行主题
> 更新日期：2026-07-22
> 保留范围：瑞士纸白、终端绿、账本蓝（生活项目平行主题，见 §7）、星图青（科研项目平行主题，见 §8）

## 1. 使用原则

1. PPT 只展示学生当场需要阅读、观察或操作的内容。教学目标、教师口令、助教分工、答案和进度调整留在教师材料中。
2. 一页聚焦一个主要问题。代码、图示和文字共同服务于课堂讲解，不把讲义整段搬上屏幕。
3. 重要程序在运行前先让学生预测；输出、报错和关键步骤可以分步揭示。
4. 风格是信息类型的视觉提示，不是装饰主题。瑞士纸白负责课程结构和叙事，终端绿负责代码与执行过程，账本蓝负责生活项目的问题情境、数据呈现与成果报告，星图青负责科研项目的任务情境、观测数据、证据结论与研究报告。
5. 一个场景用途一个主题：不为装饰换皮，也不在已有主题上叠加内容色层；新主题以完整平行主题加入，自带版式骨架、字体策略与组件体系。
6. 主题以页为最小单位整体切换：一页属于什么场景，整页就使用对应主题；不在同一页内混用多个主题的版式骨架、配色或组件。

## 2. 瑞士纸白

**建议用途**：封面、课程地图、问题定义、概念关系、环节切换、检查点和总结。

```css
--s-bg: #fafaf8;
--s-ink: #0a0a0a;
--s-accent: #ff6b35;
--s-muted: #6b6b66;
--s-hairline: rgba(10, 10, 10, 0.14);
```

- 拉丁标题可使用 `Archivo` 等高字重无衬线字体；中文标题使用清晰的宋体类字体，正文使用系统黑体类字体。
- 使用明确网格、发丝线、对位标记、章节数字和安全橙强调建立信息层级。
- 排版可以不对称，但阅读顺序必须明确，视觉重心保持稳定。
- 动效以克制的淡入和位移为主，避免与信息无关的持续运动。
- 适合呈现“为什么学、问题是什么、知识如何关联、这一段学完了什么”。
- 现有独立参考页：[`../.design/slide-previews/瑞士纸白.html`](../.design/slide-previews/瑞士纸白.html)。其中外链字体和具体布局仅供视觉参考，不能直接视为新课件交付标准。

## 3. 终端绿

**建议用途**：语法拆解、代码执行、变量追踪、Traceback 阅读、调试流程和故障速查。

```css
--b-bg: #0d1117;
--b-panel: #161b22;
--b-border: #30363d;
--b-green: #39d353;
--b-text: #c9d1d9;
--b-muted: #8b949e;
--b-red: #ff7b72;
```

- 代码使用 `JetBrains Mono` 等等宽字体；中文使用可读性良好的系统字体回退栈。
- 所有代码和运算符禁用字体连字，确保 `==`、`>=`、`<=`、`!=` 逐字符显示。
- 终端窗口、提示符、语法着色和行级高亮用于解释执行过程；红色只用于错误和风险信息。
- 扫描线、暗角和打字效果只能低强度使用，不能降低投影可读性。
- 代码与输出可以并排呈现；单页代码保持足够短，以便逐行讨论。
- 现有独立参考页：[`../.design/slide-previews/终端绿.html`](../.design/slide-previews/终端绿.html)。其中外链字体和具体布局仅供视觉参考，不能直接视为新课件交付标准。

## 4. 组合方式

推荐的基本节奏是：

```text
瑞士纸白：提出问题与建立结构
-> 终端绿：观察代码与执行过程
-> 瑞士纸白：抽象方法与总结迁移
```

预测、调试和代码追踪优先使用终端绿；问题情境、思维框架、方案比较和阶段总结优先使用瑞士纸白；M1–M4 生活项目的情境引入、数据呈现和成果报告使用账本蓝；M5–M8 科研项目的任务简报、观测数据、证据解释、规则比较和研究报告使用星图青。练习页不固定使用某一风格，以学生需要完成的认知任务为准。

## 5. 投影可读性

- 以 1280×720 教室投影为最低验证基准。
- 页面主标题不小于 40px，正文不小于 24px，代码不小于 26px，辅助小字不小于 18px。
- 一页正文要点不超过 4 条，代码通常不超过 8 行；超出时优先拆页。
- 主内容宽度控制在约 76vw 内，避免信息散落在屏幕四角。
- 同一页最多设置两个主要视觉焦点。
- 动效关闭或 `prefers-reduced-motion` 开启时，全部信息仍应完整可读。

## 6. 已确认的落地决策

以下事项已于 M0 PPT 样板阶段（2026-07-22）确认：

- 最终课件格式：**单文件只读 HTML**（不含浏览器内编辑器），每份 PPT 对应一个内容模块。M0 交付物为 [`../modules/00-course-intro/课程介绍与编程概述课件.html`](../modules/00-course-intro/课程介绍与编程概述课件.html)。
- 字体方案：拉丁字体内联 woff2 子集（Archivo 400/700/800、JetBrains Mono 400/700），中文使用系统字体栈（标题宋体类、正文黑体类），文件完全离线可用。
- 瑞士纸白页使用网格 + 十字对位 + 幽灵页码 + 发丝线编号行；终端绿页使用终端窗口 + 低强度扫描线与暗角；动效为进入时错峰淡入，课堂分步揭示由教师按键控制（`.build` 机制）。
- 验证基准为 1280×720 桌面投影；暂不做移动端适配。
- 发布方式：直接打开 HTML 放映，方向键 / 空格翻页，F 全屏，支持 `#页码` 直达。

像素轨道、深空火星主题和旧版三风格章节节奏不再保留为设计要求。

## 7. 平行主题 · 账本蓝（生活项目）

> 回答《贯穿式案例主题》§8 的开放问题：生活项目使用与瑞士纸白、终端绿并列的第三种完整主题“账本蓝”，服务 M1–M4「我的一周预算助手」。一场景一主题，不做叠加内容层。

**建议用途**：生活项目的问题情境引入、数据呈现、状态解释与成果报告页。代码与执行过程仍归终端绿，课程结构与阶段总结仍归瑞士纸白。

```css
--a-bg: #F4F6FB;                /* App 浅灰底 */
--a-card: #FFFFFF;              /* 白色卡片 */
--a-ink: #101828;               /* 近黑正文 */
--a-muted: #667085;             /* 辅助灰 */
--a-line: #E4E9F2;              /* 卡片分隔细线 */
--a-blue: #2F6BFF;              /* 主色 · 现代 App 蓝 */
--a-deep: #1D4ED8;              /* 深蓝（渐变/强调） */
--a-tint: #E8EFFF;              /* 浅蓝底 */
--a-ok: #12B76A;                /* 预算正常 · 仅胶囊/徽标/刻度 */
--a-warn: #F79009;              /* 接近上限 · 仅胶囊/刻度 */
--a-over: #F04438;              /* 已经超支 · 仅胶囊 */
```

- 版式骨架：浅灰底 + 顶部淡蓝渐变、顶部应用栏（logo、账期胶囊、＋记一笔按钮、模块标签）、白色圆角卡片为信息容器、右下角巨型幽灵 ¥（平行于纸白的幽灵页码）、页脚主题牌与页码。不使用瑞士纸白的网格、十字对位与安全橙，也不使用古风账簿元素（书脊、红栏、印章、楷体）。
- 卡片纪律：信息装入白卡（圆角 16–20px + 轻投影）或直接置于底上；分隔只用 #E4E9F2 细线与留白，不使用装饰性格线与描边堆叠。
- 字体策略：全无衬线 —— 标题 Archivo 800 + 系统黑体 Bold，正文系统黑体栈，金额与行内代码使用 JetBrains Mono 等宽（禁用连字）。
- 签名组件：余额卡（蓝紫渐变 + 大数字结余 + 内嵌胶囊进度条）、预算进度条（圆角胶囊 + 琥珀色 80% 提醒线刻度，对应 `spending_ratio >= 0.80`）、状态软胶囊（预算正常 / 接近预算上限 / 预算恰好用完 / 已经超支，当前状态实心，其余软底）、类别图标行（圆角方块字母 F / T / S / O + 金额等宽右对齐）、电子账单卡（标签/值行 + 虚线分隔 + ✓ 已生成徽标）、每日支出迷你柱图。
- 状态语义色只出现在状态胶囊、进度条刻度与徽标，不作页面主题色；一页内数据焦点不超过一个（余额卡或账单卡）。
- 账本蓝页内不放终端窗口；行内代码使用钢笔蓝底纹小片。
- 字号、动效与拆页规则遵循 §5；字体与离线方案遵循 §6。
- 独立参考页：[`../.design/slide-previews/账本蓝.html`](../.design/slide-previews/账本蓝.html)。其中外链字体和具体布局仅供视觉参考，不能直接视为新课件交付标准。科研项目（行星分析器）的平行主题「星图青」见 §8。

## 8. 平行主题 · 星图青（科研项目）

> 回答《贯穿式案例主题》§8 的第二个开放问题：科研项目使用与瑞士纸白、终端绿、账本蓝并列的第四种完整主题「星图青」，服务 M5–M8「群星彼岸·候选行星分析器」。一场景一主题，不做叠加内容层。旧版像素轨道与深空火星素材不再复用，本主题按新的信息类型完整设计。

**建议用途**：科研项目的任务简报（问题情境引入）、观测数据呈现、证据与结论解释、规则方案预演比较和研究报告页。代码与执行过程仍归终端绿，课程结构与阶段总结仍归瑞士纸白，生活项目情境仍归账本蓝。

```css
--p-bg: #0B1124;                /* 深空墨蓝 */
--p-panel: #141C36;             /* 档案面板 */
--p-panel-2: #1A2444;           /* 嵌层面板 / 量规轨道 */
--p-border: #2B3863;            /* 星图细线 */
--p-ink: #EAEEFB;               /* 主文字 */
--p-muted: #8E97BD;             /* 辅助灰蓝 */
--p-cyan: #41D9D6;              /* 主色 · 光谱青 */
--p-violet: #9B8CFF;            /* 辅色 · 星云紫（渐变/次要标记） */
--p-tint: rgba(65, 217, 214, 0.12);  /* 青光底纹 */
--p-ok: #34D399;                /* 优先候选 · 仅徽标/信号点/量规标记 */
--p-wait: #FBBF24;              /* 需要更多观测 · 仅徽标/信号点 */
--p-stop: #F87171;              /* 不建议继续 · 仅徽标/越界标记 */
```

- 版式骨架：深空墨蓝底 + 星场（细星层 + 亮星层）与星图经纬网格、低强度星云渐变、顶部任务栏（轨道徽标、任务名、观测批次胶囊、候选计数或按键提示、视场坐标读数、模块标签）、深色档案面板为信息容器、右下角巨型刻度盘式幽灵轨道环（平行于纸白的幽灵页码与账本的幽灵 ¥）、页脚主题牌与页码。不使用瑞士纸白的网格、十字对位与安全橙，不使用账本蓝的白卡、应用栏与 ¥ 符号，也不使用终端窗口、提示符与扫描线。
- 面板纪律：信息装入深色档案面板（圆角 16–20px + 1px 星图细线边，外发光仅用于签名档案卡）或直接置于底上；分隔只用 #2B3863 细线、虚线与留白，不堆叠装饰性描边。
- 字体策略：全无衬线 —— 标题 Archivo 800 + 系统黑体 Bold，正文系统黑体栈，观测读数、编号与行内代码使用 JetBrains Mono 等宽（禁用连字）；小标签可使用大写字母加字距的仪器风格。
- 签名组件：候选档案卡（行星名 + 取景框角标 + 结论徽标 + 指标量规 + 信号行）、指标量规（横向轨道条 + 接受区间高亮带 + 值标记 + 仪表刻度与通过状态灯，越界时标记与状态灯变红，服务「边界也算通过」教学）、辐射等级点（5 圆点，填充数即等级，超上限整组变红）、光谱信号确认条（波段尖峰 + H₂O 标签）与观测状态软片、结论徽标三态（优先候选 / 需要更多观测 / 不建议继续，当前结论实心，其余软底）、候选目录行（编号 + 名称 + 结论色点）、系统状态灯条（封面遥测汇总：规则版本、候选计数、各结论数量）、巡天星座标绘（封面装饰，候选即星点）、规则预演卡（基准 vs 保守对照 + 变化标记 + 结论变化 before → after，标注「预演在深拷贝副本中进行，基准研究不变」）、研究报告卡（报告编号 + 候选结论行 + 优先候选汇总 + 限制声明脚注）。
- 状态语义色只出现在结论徽标、信号点与量规标记，不作页面主题色；一页内数据焦点不超过一个（一张档案卡或一张报告卡）。
- 星图青页内不放终端窗口与提示符；行内代码使用青光底纹小片。
- 与终端绿同为深色主题，区分靠色相（蓝紫底 + 光谱青 vs 灰黑底 + 终端绿）、组件（档案面板 vs 终端窗口）与底纹（星点网格 vs 扫描线）；星场、经纬网格、星云与幽灵轨道环只低强度使用，亮星层闪烁保持极低强度，不降低投影可读性。
- 字号、动效与拆页规则遵循 §5；字体与离线方案遵循 §6。
- 独立参考页：[`../.design/slide-previews/星图青.html`](../.design/slide-previews/星图青.html)。其中外链字体和具体布局仅供视觉参考，不能直接视为新课件交付标准。
