## 1. 课程定位

本课程面向已高中毕业、即将进入大学的准大一新生。学生大多将学习理工科，整体学习能力和抽象推理能力较强，但课程不假设其具备编程经验。

课程不是 Python 语法条目的速览，而是以 Python 为载体的编程与计算思维入门。学生不仅需要写出能够运行的程序，还应逐步学会分析问题、设计步骤、表示数据、控制执行、定位错误和解释自己的方案。

## 2. 时间与人员

- 课程共 6 天，每天安排 2 个课时。
- 每课时 50 分钟，共 12 个课时、600 分钟（10 小时）。
- 主讲：马锦艺。
- 助教：梁健、谢雨荷，具体课时分工见排课表。
- 第 0 节是课程内容编号，安排在现有 12 个课时内，不额外增加课时。

## 4. 课程总体目标

完成课程后，学生应能够：

1. 在已安装或课堂提供的环境中，独立创建、运行和修改 Python 程序，并能按安装指南完成基本环境确认。
2. 阅读程序并按执行顺序追踪表达式、变量和控制流。
3. 将一个具体问题说明为输入、输出、规则、状态和处理步骤。
4. 使用变量、基本类型、输入输出和运算表达程序状态。
5. 使用条件判断和循环实现分支、重复、验证与累计过程。
6. 使用函数分解问题，通过参数和返回值设计清晰接口。
7. 根据问题选择列表、字典、集合、元组等数据表示方式。
8. 理解对象身份、引用、可变性、副作用以及深浅拷贝的差异。
9. 使用示例、边界情况和错误信息测试、调试并改进程序。
10. 清楚解释一个方案为何正确、如何组织以及还有哪些限制。

## 5. 编程思维主线

全课程反复使用同一套问题解决流程：

```text
明确问题
-> 找出输入和输出
-> 构造具体例子
-> 拆分处理步骤
-> 选择状态与数据表示
-> 编写代码
-> 手动追踪
-> 使用边界情况测试
-> 调试和改进
```

每个内容模块都同时设置知识目标和思维目标，并至少包含预测、追踪、独立编写、边界测试、调试和解释中的若干活动。

## 7. 贯穿式案例与实施原则

课程采用两个前后衔接的小项目组织主要实践：第 1–3 天完成“我的一周预算助手”，突出程序在日常生活中的直接用途；第 4–6 天完成“群星彼岸·候选行星分析器”，训练函数、容器、拷贝、科研证据和限制说明。第 7 课通过函数化重构与单星球分析完成领域迁移。完整主题决定和设计边界见 [`贯穿式案例主题.md`](贯穿式案例主题.md)，逐课时实施初稿见 [`贯穿式项目实施计划.md`](贯穿式项目实施计划.md)。

- 课程材料按知识模块建设，推荐授课路线按 6 天组织，但两者不强绑定。
- 两个项目分别形成完整闭环，不在每节课交替切换，也不要求共享同一个持续膨胀的代码文件。
- 实践不只指完整项目，也包括代码阅读、输出预测、手工追踪、找错、方案比较和小型编程任务。
- 生活项目以真实状态和边界反馈体现实用价值；科研项目以观测数据、可解释规则和方案隔离体现科学思维。
- 学生能力较强不等于已有编程经验；可以提高推理深度和独立探索比例，但不能省略首次操作、最小示例和必要反馈。
- “清北苗子”等内部学生画像不出现在 PPT 内容稿中。
- PPT 内容稿只承载学生当场需要看到的内容；教学目标、助教分工、时间调整和参考答案留在教师教案中。
