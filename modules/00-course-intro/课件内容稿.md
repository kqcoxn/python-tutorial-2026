## 1. 内容与呈现原则

- 每页的“屏幕内容”是学生当场需要看到的文字；“制作备注”不进入最终画面。
- 问题页与答案页分开，确保学生先预测、讨论或执行，再看到结果。
- 每页风格必须是四主题之一（瑞士纸白 / 终端绿 / 账本蓝 / 星图青），以页为最小单位整体切换；不写“过渡”“呼应两套视觉”“以某色为强调色”等混用或叠加描述。本节使用瑞士纸白（课程结构、问题、概念关系和总结）和终端绿（程序与执行结果）。
- M0 只建立编程全景和最小执行直觉，不正式讲解 Python 语法。

### 15 · 按精确步骤执行

**屏幕内容**

```text
初始状态
3   1   4   1
```

```text
1. 从最左边一对数字开始。
2. 比较相邻两个数字；左边更大时交换。
3. 向右移动一格，直到最右端。这是一轮。
4. 如果这一轮发生过交换，回到第 1 步。
5. 如果完整一轮没有交换，停止。
```

**制作备注**

- 风格：瑞士纸白。
- 数字使用固定位置的四个单元格，步骤始终保持可见。
- 不显示算法名称。

### 16 · 执行结果与检查

### 18 · 平均速度拆解

**屏幕内容**

```text
小车行驶 12 m，用时 3 s，求平均速度。
```

| 部分 | 内容                     |
| ---- | ------------------------ |
| 输入 | 路程 `12 m`，时间 `3 s`  |
| 处理 | 平均速度 = 路程 ÷ 时间   |
| 输出 | `4 m/s`                  |
| 约束 | 单位一致；时间不能为 `0` |

```text
如果只写“输入 12 和 3，输出 4”，
还缺少什么信息？
```

**制作备注**

- 风格：瑞士纸白。
- 表格内容分步出现，先保留情境和追问，再揭示完整拆解。
