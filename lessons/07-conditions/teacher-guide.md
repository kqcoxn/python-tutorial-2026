# 第 7 章教师教案：条件分支与单日资源调度

> 更新日期：2026-07-18  
> 适用学生：高一及初升高零基础混班  
> 建议用时：100-150 分钟，可拆分为独立模块  
> 项目版本：从 V5 扩展到 V6

## 1. 章节定位

第 6 章解决的是“这个条件成立吗”，本章解决的是“条件成立或不成立时，程序分别做什么”。学生要把布尔表达式变成真正影响程序执行路径的决策结构。

本章完成后，火星基地项目将从“资源条件检查器”升级为“单日资源调度器”：用户选择任务，程序只执行对应分支；科研任务还要进行第二层资源检查；有效任务结束后统一结算资源。

所有资源数值均为编程教学使用的无单位模拟值，不代表真实火星工程参数。

教学时先用成绩判断、奇偶判断、数值正负和周末安排等熟悉示例分别讲清语法。完成嵌套判断后，再明确切换到火星项目集中编写。不要在新语法首次出现时同时解释资源规则。

## 2. 学习目标与验收证据

| 学习目标 | 可观察的验收证据 |
| --- | --- |
| 使用单个 `if` 控制一段代码是否执行 | 输入空基地名时使用默认名称，非空时保留原名称 |
| 使用 `if-else` 表达二选一 | 能写出资源充足与不足的两种输出 |
| 使用 `if-elif-else` 表达互斥的多选一 | 输入 `1`、`2`、`3`、其他值时进入正确分支 |
| 根据缩进识别代码块 | 能在不运行代码时圈出每个分支控制的语句 |
| 区分多个独立 `if` 与一条分支链 | 能解释“允许多项同时发生”和“只选一项”的差别 |
| 使用嵌套判断处理二阶段决策 | 只在选择科研任务后检查资源是否充足 |
| 完成并验证单日资源调度器 | 四类输入均得到规定的资源结果 |

最低通过标准：学生不看完整参考代码，能够补全三任务分支和无效输入分支，并口头解释任意一次运行经过了哪些判断。

## 4. 核心词汇

| 词汇 | 本章含义 |
| --- | --- |
| 条件 | 结果为 `True` 或 `False` 的表达式 |
| 分支 | 根据条件选择执行的代码路径 |
| 代码块 | 由同一级缩进组织在一起的一组语句 |
| 缩进 | 行首空格；Python 用它表示代码层级 |
| 互斥 | 多个选项中一次只应执行一个 |
| 嵌套 | 在一个分支内部再进行一次判断 |
| 分支顺序 | Python 从上到下检查条件的次序 |
| 条件覆盖 | 设计输入，使每个分支都至少执行一次 |

## 7. 核心教学步骤

### 步骤 1：用传统分数示例追踪布尔结果

**目标**：把已学的布尔表达式与即将学习的执行路径连接起来。

先让学生填写预测：

```python
score = 59
passed = score >= 60
print(passed)
```

| `score` | `passed` 预测 |
| ---: | --- |
| 59 | `False` |
| 60 | `True` |
| 100 | `True` |

追问：程序虽然算出了 `passed`，但它是否已经在及格和不及格时执行了不同操作？

预期结论：布尔值只是判断结果；还需要分支结构决定接下来执行哪些语句。

### 步骤 2：用单个 `if` 输出及格提示

**目标**：理解“条件为真就执行，否则跳过”。

先投影并要求预测 59 分和 60 分：

```python
score = int(input("请输入分数："))

if score >= 60:
    print("及格")

print("判定结束")
```

板书执行模型：

```text
检查条件
├─ True  -> 执行缩进代码块 -> 继续向下
└─ False -> 跳过缩进代码块 -> 继续向下
```

必须指出：

- 条件末尾有英文冒号 `:`。
- 同一代码块通常使用 4 个空格缩进。
- `print("判定结束")` 与 `if` 对齐，所以无论条件真假都会执行。
- 单个 `if` 不要求必须有 `else`。

**跟随练习 1**：温度不低于 35 时输出“今天适合待在空调房里”；低于 35 时不输出这句话。

```python
temperature = 36

if temperature >= 35:
    print("今天适合待在空调房里。")
```

### 步骤 3：用 `if-else` 判断奇偶

**目标**：理解一次运行恰好执行两个分支中的一个。

```python
number = int(input("请输入一个整数："))

if number % 2 == 0:
    print("这是偶数。")
else:
    print("这是奇数。")
```

预测问题：

1. 输入 8 时，第二个 `print()` 会不会执行？
2. `else` 后面为什么没有条件？
3. 能否把 `else` 单独写在没有 `if` 的位置？

预期结论：`else` 表示前面的条件未成立时剩下的情况；两个代码块只执行一个。

**跟随练习 2**：输入一个整数，根据它是否大于等于 0 输出“非负数”或“负数”，分别用 0 和 -1 验证边界。

### 步骤 4：用 `if-elif-else` 判断正负与零

**目标**：理解多选一分支链从上到下检查，并且只执行一条路径。

先写传统的数值分类示例：

```python
number = int(input("请输入一个整数："))

if number > 0:
    print("正数")
elif number < 0:
    print("负数")
else:
    print("零")
```

执行规则：Python 从上到下检查；遇到第一个为 `True` 的条件后执行对应代码块，并跳过这条链剩余的分支。所有条件都不成立时才执行 `else`。

课堂追问：

- 输入 5 时会检查到哪一个条件？
- 输入 -2 时，第一个条件为 `False` 后程序做什么？
- 输入 0 时，前两个条件分别是什么结果？
- 一条分支链是否可能同时输出“正数”和“负数”？

**验证**：依次输入 5、-2、0，每次只能看到一种分类。

### 步骤 5：区分多个独立 `if` 与分支链

**目标**：根据业务含义选择结构，而不是机械地把第二个 `if` 改成 `elif`。

适合多个独立 `if` 的情况：一个数字可以同时满足多个性质。

```python
number = 6

if number % 2 == 0:
    print("能被 2 整除")

if number % 3 == 0:
    print("能被 3 整除")
```

输入 6 时两条信息都应输出，因此不能改成 `elif`。

适合 `if-elif-else` 的情况：一个整数在正数、负数、零三类中只能属于一类。

```python
number = -4

if number > 0:
    print("正数")
elif number < 0:
    print("负数")
else:
    print("零")
```

判断口令：

- 问“这些情况能不能同时发生？”
- 可以同时发生，考虑多个独立 `if`。
- 一次只能选一个，考虑一条 `if-elif-else` 分支链。

不要告诉学生“看到多个条件就用 `elif`”；结构取决于问题本身。

### 步骤 6：用轻松示例理解嵌套判断

**目标**：理解只有外层条件成立时，程序才会检查内层条件。

```python
is_weekend = input("今天是周末吗（是/否）：")

if is_weekend == "是":
    homework_done = input("作业写完了吗（是/否）：")

    if homework_done == "是":
        print("可以安心打游戏。")
    else:
        print("先写作业，再打游戏。")
else:
    print("先去上课。")
```

路径追踪：

```text
今天是否为周末？
├─ 否 -> 先去上课
└─ 是 -> 作业是否写完？
         ├─ 是 -> 可以安心打游戏
         └─ 否 -> 先写作业，再打游戏
```

强调两点：

- 内层 `if` 只在外层周末条件成立时检查。
- 嵌套是因为存在两阶段决策，不是为了让代码看起来复杂。

输入预测：

| 是否周末 | 作业是否完成 | 结果 |
| --- | --- | --- |
| 是 | 是 | 可以安心打游戏 |
| 是 | 否 | 先写作业，再打游戏 |
| 否 | 是或否 | 先去上课 |

### 步骤 7：集成单日资源调度

**目标**：按三个阶段更新变量，并让所有执行路径都可验证。

教师明确提示：“前面是独立的语法演示，现在开始把本章知识集中接回火星基地项目。”先运行 V5，回顾 `can_explore` 只产生布尔结果，再开始扩展 V6。

先给出教学规则表：

| 调度任务 | 任务直接效果 |
| --- | --- |
| `1` 维护供能 | 能源 `+25` |
| `2` 维护生命保障 | 能源 `-15`，氧气 `+20`，饮用水 `+15` |
| `3` 科研探索 | 能源 `-20`，氧气 `-10`，饮用水 `-8`，科研积分 `+30` |
| 其他输入 | 任务无效，资源不变 |

每个有效任务完成后还要结算日常消耗：能源 `-5`、氧气 `-10`、饮用水 `-8`。三项基础资源上限均为 100。上述规则是教学模型，不是现实工程数据。

让学生按以下顺序扩展，而不是一次输入完整 V6：

1. 从 V5 另存为新文件，建立基地名、四项资源和 `task_completed`。
2. 加入菜单与任务编号输入。
3. 先完成任务 `1`、任务 `2` 和无效输入分支。
4. 在任务 `3` 中嵌套 `can_explore` 判断。
5. 只有 `task_completed` 为 `True` 时结算日常消耗。
6. 用三个独立 `if` 将超出 100 的资源分别限制为 100。
7. 输出基地名、三项资源和科研积分。

这里三个资源上限使用独立 `if`，因为能源、氧气和饮用水可能分别需要限制，不是三选一。

### 步骤 8：用手算表验证程序

**目标**：区分“程序能运行”和“程序结果正确”。

初始资源均为 100，科研积分为 0。每组输入都重新运行程序：

| 输入 | 任务效果后 | 日常消耗后与上限处理后 | 最终科研积分 |
| --- | --- | --- | ---: |
| `1` | 能源 125，氧气 100，水 100 | 能源 100，氧气 90，水 92 | 0 |
| `2` | 能源 85，氧气 120，水 115 | 能源 80，氧气 100，水 100 | 0 |
| `3` | 能源 80，氧气 90，水 92 | 能源 75，氧气 80，水 84 | 30 |
| `9` | 任务无效 | 能源 100，氧气 100，水 100 | 0 |

要求学生在运行前先写下结果。若结果不同，按“选中分支 -> 任务更新 -> 日常消耗 -> 上限处理”的顺序逐段检查。

## 8. 常见误解与讲解回应

| 学生表现 | 实际问题 | 教师回应 |
| --- | --- | --- |
| `if choice = "1":` | 把赋值与比较混淆 | 追问“这行是在改变 choice，还是提出一个真假问题？” |
| `if choice == 1:` 无法进入分支 | `input()` 返回字符串 | 打印 `type(choice)`，再改为与 `"1"` 比较 |
| 条件行报 `SyntaxError` | 常见为末尾缺少冒号 | 从报错行末尾检查 `:`，再检查上一行括号 |
| `elif` 或 `else` 报错 | 没有与所属 `if` 对齐 | 用竖线标出三者必须处于同一缩进层级 |
| 分支内只有第一行受控制 | 后续语句没有保持相同缩进 | 让学生圈出该分支应包含的全部动作 |
| 输入 `1` 后又显示“无效选择” | 使用了多个 `if`，`else` 只绑定最近的 `if` | 将互斥任务改成一条 `if-elif-else` 链 |
| 90 分显示普通完成 | 宽条件写在窄条件之前 | 先检查 `>= 90`，再检查 `>= 60` |
| 每次任务都扣除或都不扣除日常消耗 | `if task_completed` 的缩进层级错误 | 先说明哪些路径算“完成一天”，再调整代码块 |
| 写成 `if can_explore == True` | 语义重复但不是语法错误 | 接受后简化为 `if can_explore:`，说明布尔变量本身就是条件 |

## 9. 找错任务与答案

### 找错 1：缺少语法符号

```python
if oxygen < 30
    print("氧气预警")
```

答案：条件末尾缺少英文冒号。

```python
if oxygen < 30:
    print("氧气预警")
```

### 找错 2：赋值与比较混淆

```python
if choice = "1":
    print("维护供能系统")
```

答案：条件中需要比较，使用 `==`。

```python
if choice == "1":
    print("维护供能系统")
```

### 找错 3：任务 1 后又显示无效

```python
if choice == "1":
    print("维护供能系统")

if choice == "2":
    print("维护生命保障系统")
else:
    print("无效选择")
```

答案：第二个 `if` 是新的判断，`else` 只与它配对。互斥任务应使用一条分支链。

```python
if choice == "1":
    print("维护供能系统")
elif choice == "2":
    print("维护生命保障系统")
else:
    print("无效选择")
```

### 找错 4：高分分支永远无法到达

```python
if research_points >= 60:
    print("任务顺利完成")
elif research_points >= 90:
    print("卓越科研任务")
```

答案：90 同时满足 `>= 60`，会被第一个分支提前接住。更具体、更严格的条件应放在前面。

```python
if research_points >= 90:
    print("卓越科研任务")
elif research_points >= 60:
    print("任务顺利完成")
```

### 找错 5：只有科研任务扣除日常消耗

```python
elif choice == "3":
    research_points += 30
    energy -= 5
    oxygen -= 10
    water -= 8
```

答案：日常消耗属于所有有效任务，不属于科研分支。应在整条任务分支链结束后，通过 `if task_completed:` 统一结算。

if research_points >= 90:

### 挑战任务：构造最小分支覆盖集

要求学生说明至少需要哪些输入，才能让单日调度器的外层每个分支都执行一次，并让科研内层的成功和失败也各执行一次。

参考答案：外层至少需要 `1`、`2`、`3`、一个无效值；科研内层还需两组资源状态，例如：

- `choice == "3"`，资源为 20、15、10，覆盖成功分支。
- `choice == "3"`，资源为 19、100、100，覆盖失败分支。

说明：因为程序每次只选择一项任务，外层四条路径不能由一次运行全部覆盖。

## 11. 预测问题答案

1. 分数 59 时只输出“判定结束”；分数 60 时先输出“及格”，再输出“判定结束”。
2. 奇偶判断一次只执行一个代码块；8 进入偶数分支，7 进入奇数分支。
3. 输入 -2 时，先得到 `number > 0` 为 `False`，再由 `number < 0` 进入负数分支。
4. 输入 0 时，`number > 0` 和 `number < 0` 都为 `False`，因此执行 `else`。
5. `number = 6` 时，两个独立 `if` 都成立，会同时输出“能被 2 整除”和“能被 3 整除”。
6. 周末且作业完成时进入内层成功分支；周末但作业未完成时进入内层 `else`；非周末时不检查作业条件。
7. 项目评级时必须先判断 `>= 90`，否则 90 分会先满足 `>= 60`。
8. 项目中的科研判断应嵌套在任务 `3` 中，因为其他任务不需要检查科研门槛。

## 13. 进度调整

### 进度快

- 先手算再测试 19/20、14/15、9/10 三组科研边界。
- 完成“无法到达的评级分支”找错题，并解释分支顺序。
- 为程序设计一项新任务，写清任务效果、是否需要二次检查和预期输出。
- 设计最小测试集，证明所有分支都被执行过。

### 进度慢

- 教师直接提供标题、变量初始化、菜单和最终输出，只让学生完成分支区。
- 暂不实现基地名默认值和资源上限，先保证三任务与无效输入正确。
- 科研分支先只输出成功或失败，确认嵌套结构后再加入资源更新。
- 找错题减少到“缺冒号”和“多个 `if`”两题。

无论进度如何，都不能省略学生独立完成一段分支代码和使用多组输入验证的环节。
