## 1. 章节定位

本章补齐固定次数循环和循环内的流程控制。学生应学会根据问题选择 `for` 或 `while`，并准确解释 `break`、`continue` 影响哪一层、哪一轮。前半段使用数字序列、三次尝试和奇数筛选；后半段集中完成最终参考结构 V8。

## 4. 核心词汇

| 词汇 | 含义 |
| --- | --- |
| `for` 循环 | 依次取得一组值并执行循环体 |
| `range()` | 生成整数范围供循环使用 |
| 左闭右开 | 包含起点，不包含终点 |
| `break` | 立即结束最近一层循环 |
| `continue` | 跳过本轮剩余代码，进入下一轮 |

## 6. 核心教学步骤

### 步骤 1：第一个 `for` 循环

```python
for number in range(5):
    print(number)
```

预期输出 0、1、2、3、4。`range(5)` 从 0 开始，不包含 5。

### 步骤 2：三种 `range()` 形式

```python
for number in range(2, 6):
    print(number)
```

输出 2、3、4、5。

```python
for number in range(10, 3, -2):
    print(number)
```

输出 10、8、6、4。

| 写法 | 开始 | 结束边界 | 步长 | 实际序列 |
| --- | ---: | ---: | ---: | --- |
| `range(5)` | 0 | 5，不包含 | 1 | 0, 1, 2, 3, 4 |
| `range(2, 6)` | 2 | 6，不包含 | 1 | 2, 3, 4, 5 |
| `range(10, 3, -2)` | 10 | 3，不包含 | -2 | 10, 8, 6, 4 |

不使用 `list(range(...))`，避免提前引入列表。

### 步骤 3：选择 `for` 还是 `while`

- 已知重复次数或整数范围：优先考虑 `for`。
- 不知道要尝试几次，只知道继续条件：优先考虑 `while`。

例：输出 7 次自检适合 `for`；输入到合法为止适合 `while`。两者有时都能实现，但应选择最直接表达题意的结构。

### 步骤 4：使用 `break` 提前结束

```python
secret = "python"

for attempt in range(1, 4):
    answer = input(f"第 {attempt} 次尝试：")

    if answer == secret:
        print("回答正确")
        break

    print("回答错误")
```

输入第一次就正确，只执行一轮；第三次才正确，执行三轮；三次都错则自然结束。`break` 退出的是 `for`，程序仍可继续执行循环后的代码。

### 步骤 5：使用 `continue` 跳过本轮

```python
for number in range(1, 8):
    if number % 2 == 0:
        continue

    print(number)
```

偶数轮执行到 `continue` 后跳过 `print()`，最终输出 1、3、5、7。

对比：`break` 是“不再循环”，`continue` 是“本轮到此，下一轮继续”。

### 步骤 6：`continue` 与 `while` 的死循环风险

```python
count = 0

while count < 5:
    if count == 2:
        continue

    count += 1
```

当 `count == 2` 时更新被跳过，程序会持续停在 2。修复可以在 `continue` 前更新，或重组条件避免跳过必要状态更新。

### 步骤 7：嵌套循环中的 `break`

```python
for row in range(1, 4):
    for column in range(1, 5):
        if column == 3:
            break
        print(row, column)
```

每一行只输出第 1、2 列；`break` 退出内层循环，外层仍继续下一行。

提醒：`continue` 之前不能执行日常消耗和任务日更新；`break` 后仍会执行循环外的最终报告。

## 7. 常见误解与排查

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| `range(1, 5)` 期待包含 5 | 忘记终点不包含 | 写出实际序列再编码 |
| 步长方向与边界不匹配 | 正步长却从大到小 | 根据方向选择正负步长 |
| `break` 后以为程序结束 | 混淆循环与程序 | 找到循环结束后的下一行 |
| `continue` 后任务日仍变化 | 更新代码放在 `continue` 前 | 无效路径尽早继续 |
| `while` 在 `continue` 处死循环 | 必要更新被跳过 | 调整更新位置或条件结构 |
| 嵌套 `break` 未退出外层 | `break` 只退出最近一层 | 通过外层状态变量控制更高层 |
| 无效输入扣资源 | 日常消耗在输入验证之前 | 只在有效任务后结算 |
