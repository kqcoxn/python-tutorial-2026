# 第 10 章学生讲义：综合作业、自由完善与展示

> 本章不学习新语法。目标是把你的程序变成经过测试、能够解释、适合展示的最终作品。

## 1. 最终交付物

- [ ] `mars_base_final.py`
- [ ] 至少四组测试记录
- [ ] 一条真实 Debug 记录
- [ ] 一项自己能够解释的完善
- [ ] 一张 3 分钟展示提纲

不要提交不能解释的整段复制代码。最终程序只使用本阶段已经学习的顺序、输入输出、变量、运算、分支和循环。

## 2. 创建 Final 文件

1. 保留已经能运行的 V8。
2. 复制并命名为 `mars_base_final.py`。
3. 第一次运行，确认结果与 V8 相同。
4. 后续完善只修改 Final 文件。

作品定位：简化的火星基地工程决策模型。资源是无单位教学模拟值，不是真实航天数据，也不是 AI 调度。

## 3. 核心验收

- [ ] 有清晰标题和任务目标。
- [ ] 用户能够输入基地名称或任务选择。
- [ ] 至少五个含义明确的状态变量。
- [ ] 使用数值、字符串和布尔值。
- [ ] 使用 f-string 显示状态。
- [ ] 使用运算更新资源。
- [ ] 使用 `if-elif-else`。
- [ ] 使用至少一种循环。
- [ ] 合理使用 `break` 或 `continue`。
- [ ] 有任务完成和应急撤离结果。
- [ ] 程序在测试输入下能够正常结束。

未全部通过时，先修复核心行为，不增加新功能。

## 6. 综合练习：30 分钟自由完善

只选择一项主改进。先填计划，再写代码。

### 可选方向

1. 改进标题、状态面板、变量名和注释。
2. 增加第四种调度决策，并写清资源规则。
3. 增加由固定任务日或资源条件触发的可复现事件。
4. 增加更清楚的输入提示、资源变化或边界反馈。

不使用随机事件、导入、列表、字典、函数、类、文件或异常处理。

## 7. 完善过程检查

### 前 5 分钟

- [ ] 写清规则。
- [ ] 写清会修改的变量和位置。
- [ ] 写出至少一组预期结果。

### 中间 20 分钟

- [ ] 每完成一小段立即运行。
- [ ] 一次只解决一个问题。
- [ ] 不破坏原有 T1、T3、T4、T5。

### 最后 5 分钟

- [ ] 停止新增功能。
- [ ] 保存文件。
- [ ] 重新运行基线测试和新功能测试。
- [ ] 准备展示提纲。
