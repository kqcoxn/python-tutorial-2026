# 第 6 章教师教案：布尔值、比较与逻辑运算

> 更新日期：2026-07-18

## 1. 章节定位

本章只解决“条件是否成立”，暂不决定成立后执行什么。学生需要能计算、组合并口头解释布尔表达式。前半段使用分数、门票和天气等普通示例，后半段集中制作 V5 资源条件检查器，为第 7 章分支结构准备。

| 学习目标 | 验收证据 |
| --- | --- |
| 使用六种比较运算符 | 能预测边界值比较结果 |
| 识别布尔值 | 能区分 `True` 与 `"True"` |
| 使用 `and`、`or`、`not` | 能用自然语言准确解释成立条件 |
| 读取真值表 | 能完成二元逻辑结果表 |
| 直观理解短路 | 能说明何时后半个条件不必计算 |
| 完成 V5 | 正确输出科研、维护、安全和风险四个布尔状态 |

最低标准：学生能独立写出一个包含三个比较和两个 `and` 的表达式，并用边界值验证。

## 4. 核心词汇

| 词汇 | 含义 |
| --- | --- |
| 布尔值 | 只有 `True` 和 `False` 两种值 |
| 比较表达式 | 比较两个值并得到布尔结果 |
| 逻辑运算 | 组合或反转布尔条件 |
| 边界值 | 条件刚好成立或刚好不成立的值 |
| 短路求值 | 结果已确定时不再计算后续部分 |

## 6. 核心教学步骤

### 步骤 1：六种比较运算符

```python
score = 60

print(score == 60)
print(score != 60)
print(score > 60)
print(score < 60)
print(score >= 60)
print(score <= 60)
```

| 运算符 | 含义 |
| --- | --- |
| `==` | 相等 |
| `!=` | 不相等 |
| `>` / `<` | 大于 / 小于 |
| `>=` / `<=` | 大于等于 / 小于等于 |

重点比较 `>` 与 `>=` 在边界 60 上的差别，并再次区分赋值 `=` 与比较 `==`。

### 步骤 2：字符串也可以比较是否相等

```python
answer = input("请输入暗号：")
is_correct = answer == "python"
print(is_correct)
```

输入 `python` 为 `True`，输入 `Python` 为 `False`。字符串比较区分大小写和空格。

### 步骤 3：`and` 要求全部成立

```python
age = 16
has_ticket = True
can_enter = age >= 12 and has_ticket

print(can_enter)
```

口头表达：“年龄至少 12，并且有票，才可以进入。”不要只念符号。

### 步骤 4：`or` 要求至少一个成立

```python
is_weekend = False
is_holiday = True
can_sleep_late = is_weekend or is_holiday

print(can_sleep_late)
```

口头表达：“周末或节假日，至少满足一个就可以晚起。”说明这里的 `or` 是包含“两者都真”的逻辑或。

### 步骤 5：`not` 反转布尔结果

```python
is_raining = False
can_go_out = not is_raining

print(can_go_out)
```

`not False` 得到 `True`，`not True` 得到 `False`。建议给复杂表达式加括号或先保存中间布尔变量，以提高可读性。

### 步骤 6：真值表与短路直觉

| A | B | `A and B` | `A or B` |
| --- | --- | --- | --- |
| `True` | `True` | `True` | `True` |
| `True` | `False` | `False` | `True` |
| `False` | `True` | `False` | `True` |
| `False` | `False` | `False` | `False` |

短路直觉：`and` 左侧为 `False` 时结果已是 `False`；`or` 左侧为 `True` 时结果已是 `True`。核心阶段只要求理解“后面可能不再计算”，不把副作用技巧作为写法推荐。

### 步骤 7：切换到项目 V5

从 V4 另存为 `v5_boolean_logic.py`，输入三项资源并计算：

```python
can_explore = energy >= 20 and oxygen >= 15 and water >= 10
needs_life_support = oxygen < 30 or water < 25
system_safe = energy > 0 and oxygen > 0 and water > 0
system_at_risk = not system_safe
```

让学生逐句翻译，不先运行：

- 科研需要三项资源全部达到门槛。
- 氧气或饮用水至少一项偏低就需要维护。
- 三项资源都大于 0 才安全。
- 风险状态与安全状态相反。

### 步骤 8：用输入覆盖真假结果

| 能源 | 氧气 | 水 | 可科研 | 需维护生命保障 | 安全 | 风险 |
| ---: | ---: | ---: | --- | --- | --- | --- |
| 20 | 15 | 10 | `True` | `True` | `True` | `False` |
| 50 | 40 | 30 | `True` | `False` | `True` | `False` |
| 0 | 100 | 100 | `False` | `False` | `False` | `True` |

这些是无单位教学阈值，不是现实工程标准。

## 7. 常见误解与排查

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 条件中写 `=` | 混淆赋值和比较 | 相等判断改用 `==` |
| `"True"` 被当作布尔值 | 引号使其成为 `str` | 使用无引号的 `True` |
| 边界 20 判断失败 | 写成 `>` 而不是 `>=` | 根据“至少”选择 `>=` |
| `and`、`or` 含义反了 | 没有翻译业务条件 | 用“全部”和“至少一个”复述 |
| `not` 作用范围不清 | 表达式过长 | 加括号或先保存中间变量 |
| 输入 `Python` 不匹配 | 字符串比较区分大小写 | 对照要求输入完全一致 |

## 8. 练习与答案

### 跟随练习 1

判断分数是否位于 0 到 100：

```python
is_valid = score >= 0 and score <= 100
```

测试 -1、0、100、101。

### 跟随练习 2

有会员或优惠券即可享受折扣：

```python
has_discount = is_member or has_coupon
```

### 独立练习 1

写出“等级至少 10，并且金币至少 50”的解锁条件。

### 独立练习 2

构造一组 V5 输入，使 `can_explore` 为 `False`，但只差饮用水一项。

参考：能源 20、氧气 15、饮用水 9。

### 找错

```python
can_play = age >= 12 or has_ticket
```

若规则要求“年龄达标且有票”，应把 `or` 改为 `and`。

### 挑战

不运行程序，分别构造让 V5 四个布尔变量呈现尽可能多组合的资源输入，并说明哪些组合由于逻辑关系不可能出现。
