## 1. 章节定位

本章让程序在条件成立时重复执行。教学主线是“初始状态 -> 循环条件 -> 循环体 -> 状态更新 -> 再检查”，而不是只背 `while` 语法。前半段使用计数、倒计时、输入验证和坐标网格；后半段集中把任务日循环和输入验证循环加入 V7。

## 4. 核心词汇

| 词汇 | 含义 |
| --- | --- |
| 循环 | 重复执行一段代码 |
| 循环条件 | 每轮开始前决定是否继续的布尔表达式 |
| 循环变量 | 影响循环条件、会在循环中更新的状态 |
| 迭代 | 循环体的一次执行 |
| 死循环 | 条件一直为真、无法按设计结束的循环 |
| 嵌套循环 | 一个循环体中包含另一个循环 |

## 6. 核心教学步骤

### 步骤 1：第一个可结束计数循环

```python
count = 1

while count <= 5:
    print(count)
    count += 1

print("循环结束")
```

逐轮表：

| 检查前 `count` | 条件 | 输出 | 更新后 |
| ---: | --- | ---: | ---: |
| 1 | `True` | 1 | 2 |
| 2 | `True` | 2 | 3 |
| 3 | `True` | 3 | 4 |
| 4 | `True` | 4 | 5 |
| 5 | `True` | 5 | 6 |
| 6 | `False` | 无 | 结束 |

强调循环结束时 `count` 是 6，而不是 5。

### 步骤 2：倒计时与更新方向

```python
remaining = 3

while remaining > 0:
    print(remaining)
    remaining -= 1

print("开始！")
```

追问：若误写 `remaining += 1`，条件会怎样变化？学生必须在运行前判断可能死循环。

### 步骤 3：认识和停止死循环

```python
count = 1

while count <= 5:
    print(count)
```

这段代码忘记更新 `count`。演示 PyCharm Stop 按钮；停止后不要盲目重启，先找到条件涉及的变量及其更新位置。

排查四问：

1. 循环条件是什么？
2. 哪些变量影响条件？
3. 它们在循环中是否更新？
4. 更新方向是否让条件接近 `False`？

### 步骤 4：使用 `while` 验证输入

```python
answer = input("是否继续（y/n）：")

while answer != "y" and answer != "n":
    print("输入无效，请重新输入。")
    answer = input("是否继续（y/n）：")

print(f"你的选择：{answer}")
```

让学生解释为什么中间使用 `and`。输入 `x`、`?`、`y`，前两次继续循环，`y` 使条件为 `False`。

### 步骤 5：嵌套循环与总次数

```python
row = 1

while row <= 3:
    column = 1

    while column <= 4:
        print(f"第 {row} 行，第 {column} 列")
        column += 1

    row += 1
```

外层 3 次，内层每次 4 次，总输出 12 行。关键点：`column = 1` 必须放在外层循环内部，使每一行都从第 1 列重新开始。

### 步骤 6：切换到项目 V7

从 V6 另存为 `v7_while_loops.py`，按以下顺序扩展：

1. 建立 `mission_day = 1`、`max_days = 7`、`mission_active = True`、`completed_days = 0`。
2. 使用外层循环：

```python
while mission_day <= max_days and mission_active:
    # 显示状态、输入任务、更新资源
```

3. 在每天内部使用输入验证循环：

```python
while choice != "1" and choice != "2" and choice != "3":

    choice = input("请输入任务编号（1/2/3）：")
```

4. 只有有效且完成的任务才扣除日常消耗、增加 `completed_days`。
5. 资源耗尽时将 `mission_active = False`。
6. 资源仍安全时增加 `mission_day`。

本章不使用 `break`、`continue`。如果科研资源不足，`task_completed = False`，外层循环会在同一个任务日重新开始。

## 7. 常见误解与排查

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 一直输出同一个数 | 忘记更新循环变量 | 在循环体内加入正确更新 |
| 数字越来越远离结束 | 更新方向错误 | 对照条件判断应增还是减 |
| 循环一次都不执行 | 初始条件已是 `False` | 手算第一次条件 |
| 少一次或多一次 | `<` 与 `<=` 混淆 | 写出期望包含的最后值 |
| 输入验证无法结束 | 条件逻辑或重新输入位置错误 | 检查 `and` 与循环内赋值 |
| 网格只输出第一行 | 内层计数器未在每轮外层重置 | 将 `column = 1` 放入外层循环 |
| 任务日不增加 | 更新被放进错误分支 | 只在完成有效任务后更新 |
| 无效输入消耗任务日 | 结算代码缩进错误 | 保证内层验证结束后才进入任务结算 |

### 找错

```python
number = 1
while number <= 3:
    print(number)
number += 1
```

答案：更新未缩进到循环体内，会死循环。
