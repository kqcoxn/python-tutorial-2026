## 1. 章节定位

本章把静态程序变成交互程序。重点是 `input()` 始终返回字符串、数值转换、算术运算和状态更新。前半段使用问候、零钱和整数计算器等普通示例，后半段集中完成 V4 能源计算器。

## 6. 核心教学步骤

### 步骤 1：接收文字输入

```python
name = input("你叫什么名字？")
print(f"你好，{name}！")
print(type(name))
```

无论输入“小林”还是 `18`，`type(name)` 都显示 `str`。输入提示文字应说明用户需要输入什么。

### 步骤 2：把数字文本转换为数值

```python
age = int(input("请输入年龄："))
height = float(input("请输入身高（米）："))

print(type(age))
print(type(height))
```

`int("18")` 得到整数 18；`float("1.75")` 得到浮点数。`str(18)` 得到字符串 `"18"`。不要把转换理解为“修改原字符串”，它产生一个转换后的值。

### 步骤 3：常见算术运算符

```python
a = 17
b = 5

print(a + b)
print(a - b)
print(a * b)
print(a / b)
print(a // b)
print(a % b)
print(a ** 2)
```

| 运算符 | 含义 | `17` 与 `5` 的结果 |
| --- | --- | ---: |
| `+` | 加 | 22 |
| `-` | 减 | 12 |
| `*` | 乘 | 85 |
| `/` | 普通除法 | 3.4 |
| `//` | 整除 | 3 |
| `%` | 余数 | 2 |
| `**` | 幂 | `17 ** 2 = 289` |

强调 `//` 是向下取整意义的整除；本章核心练习使用非负整数，负数整除细节可作挑战。

### 步骤 4：运算顺序与括号

```python
result_a = 2 + 3 * 4
result_b = (2 + 3) * 4

print(result_a)
print(result_b)
```

预期为 14 和 20。教学策略：不要求背长表，先遵循括号、幂、乘除、加减；表达意图不清时主动加括号。

### 步骤 5：复合赋值更新状态

```python
coins = 10
coins += 5
coins -= 3
print(coins)
```

展开解释：

```text
coins += 5  等价于  coins = coins + 5
```

预期最终为 12。区分 `=`、`+=` 和以后用于比较的 `==`。

### 步骤 6：普通输入计算器

```python
price = float(input("请输入单价："))
quantity = int(input("请输入数量："))
total = price * quantity

print(f"总价：{total}")
```

使用单价 8.5、数量 4，预期总价 34.0。若输入“four”，观察 `ValueError`，按统一排错流程判断 `int()` 收到了不能解释为整数的文本。

## 7. 常见误解与排查

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| `"2" + "3"` 得到 `"23"` | 输入仍是字符串 | 在计算前使用 `int()` 或 `float()` |
| `TypeError`：字符串和整数相减 | 未转换输入 | 检查 `type()`，再转换 |
| `ValueError` | `int()` 收到非整数字符串 | 使用合法数字重新运行；异常处理后续学习 |
| `/` 与 `//` 结果不同 | 混淆普通除法和整除 | 写出期望“商”还是“完整小数” |
| `energy =- 20` 结果异常 | 写成赋值负数而不是 `-=` | 改为 `energy -= 20` |
| 结果优先级不符 | 缺少括号或预测错误 | 分步写出括号、乘除、加减 |
| 使用 `eval(input())` | 不安全的输入执行 | 改用明确的 `int()` 或 `float()` |
