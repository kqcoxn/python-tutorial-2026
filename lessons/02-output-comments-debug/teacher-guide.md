## 1. 章节定位

本章建立学生的第一套排错流程。目标不是记住所有错误名称，而是养成“先读信息、再提出原因、一次只改一处”的习惯。前半段使用普通最小程序认识输出、注释和错误；后半段再集中修复火星项目 V1。

## 2. 学习目标与验收证据

| 学习目标 | 验收证据 |
| --- | --- |
| 使用多次 `print()` 和多个参数输出 | 能预测输出行数与参数间空格 |
| 使用 `#` 编写有效注释 | 注释说明代码目的，且不会出现在输出中 |
| 区分代码、注释、输出和错误信息 | 能在屏幕上指出四者位置 |
| 阅读基础 Traceback | 能说出错误类型、行号和一个可能原因 |
| 使用断点和单步执行 | 能暂停程序并逐行继续 |
| 修复 V1 故障版本 | 修复后输出三条自检与任务简报 |

最低标准：学生能独立修复一个 `SyntaxError` 和一个 `NameError`，并说明自己根据哪一条信息定位问题。

## 4. 核心词汇

| 词汇 | 含义 |
| --- | --- |
| 注释 | 供人阅读、不会作为代码执行的说明 |
| Bug | 程序中的错误或不符合预期的行为 |
| Debug | 定位、解释、修复并验证问题的过程 |
| Traceback | Python 输出的错误追踪信息 |
| 断点 | 让程序在指定行执行前暂停的标记 |
| 单步执行 | 暂停后一次执行一行代码 |

## 6. 核心教学步骤

### 步骤 1：预测多次输出与多个参数

```python
print("答案是", 42)
print("不要慌")
```

运行前追问：输出有几行？第一行两个参数之间是否需要自己输入空格？

预期输出：

```text
答案是 42
不要慌
```

只说明 `print()` 可以接收多个内容，默认用空格分隔；`sep` 和 `end` 留作挑战，不作为核心要求。

### 步骤 2：区分注释和代码

```python
# 下面一行负责显示程序标题。
print("今天也要写出能运行的代码")

# TODO：以后再让程序自动写作业。
```

提问：三行文字中哪些会显示？删除注释会不会改变程序功能？

强调：注释解释“为什么这样做”或标记关键步骤，不是逐行翻译代码。三引号字符串不作为“多行注释”教授。

### 步骤 3：认识四类基础错误

分别运行，不要一次投影全部答案。

```python
print("少了右括号"
```

常见结果：`SyntaxError`。代码结构不符合语法，程序通常无法开始正常执行。

```python
pritn("拼写也很重要")
```

常见结果：`NameError`。Python 不认识 `pritn` 这个名称。

```python
    print("不该出现的缩进")
```

常见结果：`IndentationError: unexpected indent`。

```python
print(10 / 0)
```

常见结果：`ZeroDivisionError`。语法正确，但执行时发生不允许的运算。

不要让学生背完整英文句子。要求其抓住错误类型和最接近自己代码的行号。

### 步骤 4：使用统一排错流程

板书并全程复用：

```text
停下来 -> 读最后一行 -> 找错误类型 -> 找行号
-> 检查该行及上一行 -> 提出一个原因 -> 只改一处 -> 再运行
```

教师演示时必须说出推理过程，例如：“错误类型是 `NameError`，指向第 1 行；我先检查名称拼写，只修改 `pritn`。”

### 步骤 5：第一次使用断点

```python
print("第 1 步：准备")
print("第 2 步：执行")
print("第 3 步：完成")
```

操作顺序：

1. 在第 2 行左侧单击设置断点。
2. 选择 Debug，而不是 Run。
3. 观察程序在第 2 行执行前暂停。
4. 使用 Step Over 逐行执行。
5. 使用 Resume 继续到结束。
6. 再次单击断点将其移除。

解释断点是“暂停观察”，不是错误本身。在线环境或 IDLE 无法完成相同操作时，用逐行增加临时 `print()` 作为备用观察方式。

### 步骤 7：V1 故障修复

提供下面的故障片段：


学生必须先记录：

1. 第一条错误类型和行号。

## 7. 常见误解与排查

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 注释出现在输出里 | 把 `#` 写在字符串引号中 | 将说明移到引号外并置于 `#` 后 |
| 整段代码变灰且不执行 | 行首误加 `#` | 只取消需要执行行的注释 |
| 只看红字第一行 | 忽略最具体的错误类型 | 从 Traceback 最后一行开始读 |
| 修改很多行后更乱 | 未提出单一原因 | 撤回无关修改，一次验证一个假设 |
| Debug 直接运行完 | 断点未设置或运行了其他文件 | 确认红点、文件名和 Debug 配置 |
| 程序停住以为死机 | 正停在断点 | 查看当前高亮行，Step Over 或 Resume |
