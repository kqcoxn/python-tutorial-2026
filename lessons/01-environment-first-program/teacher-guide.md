# 第 1 章教师教案：环境、Python 与第一个程序

> 更新日期：2026-07-18  
> 适用学生：高一及初升高零基础混班  
> 建议用时：课前安装 + 课堂 45-75 分钟  
> 项目版本：V0，完成后可进入 V1

## 1. 章节定位

本章的任务不是详细介绍 Python 历史，也不是把 PyCharm 的所有按钮讲一遍。学生需要建立第一个完整操作闭环：

```text
找到代码文件 -> 输入代码 -> 运行 -> 找到输出 -> 修改 -> 再次运行
```

软件安装应尽量在课前完成。课堂中的环境处理只服务于“让学生尽快开始写代码”，不能让全班长时间等待个别设备。

## 2. 学习目标与验收证据

| 学习目标 | 可观察的验收证据 |
| --- | --- |
| 区分 Python、解释器、`.py` 文件和 PyCharm | 能用自己的话说明四者关系 |
| 找到本机 Python 解释器 | 能运行版本检查命令，或在 PyCharm 中选中 Python 3.x |
| 创建并运行 Python 文件 | 能从空白项目创建 `main.py` 并看到输出 |
| 修改程序并重新运行 | 能修改输出文字，运行结果随之改变 |
| 识别基础环境问题 | 能指出代码区、运行按钮和输出区的位置 |

最低通过标准：学生独立创建并运行下面的代码，而不是只观看教师演示。

```python
print("Hello, Python!")
```

## 3. 开始条件

学生不需要具备编程经验。开始前只要求：

- 能够进行基本的文件操作和键盘输入。
- 知道自己使用 Windows 还是 macOS。
- 设备电量充足，并携带充电器。

教师需要提前获得班级设备情况，至少区分：Windows、macOS Apple Silicon、macOS Intel，以及是否存在没有管理员权限的设备。

## 4. 课前准备

### 4.1 教师材料

- [ ] 在 Windows 11 普通账户上完整安装并验证一次。
- [ ] 在 macOS Apple Silicon 上完整安装并验证一次。
- [ ] 准备两套内容相同的 U 盘离线安装包。
- [ ] U 盘中包含 Python、PyCharm、安装说明和 `v0_first_program.py`。
- [ ] 准备 [Online Python](https://www.online-python.com/) 的短链接或二维码。
- [ ] 准备一台备用电脑，或明确同伴结对方案。
- [ ] 投影前关闭个人通知，放大 PyCharm 编辑器和运行区字体。
- [ ] 将本章 V0、V1 文件放在容易打开的位置。
- [ ] 投影调试本章配套幻灯片 `slides.html`：确认键盘翻页、第 5/9 页点击揭示互动和激光笔可用。

正式上课前 3 到 7 天，要求学生提交以下任一结果：

1. PyCharm 成功运行 `print("Hello, Python!")` 的截图。
2. IDLE 成功运行相同代码的截图。
3. 无法安装时提交操作系统、错误信息和截图。

### 4.2 版本策略

课程前两周冻结一套已验证的稳定版本。不要在上课当天临时更新 Python 或 PyCharm。

- Python：使用 python.org 提供的稳定 Python 3 版本。
- PyCharm：使用 JetBrains 官方安装包及免费可用功能。
- 安装包：只从官方来源下载，不使用第三方打包站或安装脚本。

本章不讲虚拟环境原理。PyCharm 自动创建项目环境时，只说明“这个项目使用哪个 Python 解释器”，把虚拟环境留到后续课程。

## 5. 主讲与助教分工

### 主讲教师

- 控制全班共同进度，只处理共性问题。
- 演示 Python、解释器、文件和 IDE 的关系。
- 每完成一个关键步骤进行全班检查。
- 任何单机故障处理不超过 2 到 3 分钟。

### 助教

- 开课前按 Windows、macOS 分区巡查。
- 优先处理解释器缺失、权限、路径和运行配置问题。
- 记录重复问题，达到 3 人时立即报告主讲统一说明。
- 环境暂时无法修复时，帮助学生切换 IDLE、在线环境或结对方案。

### 学生状态信号

每个检查点让学生用手势或桌面卡片报告：

- 绿色：完成，可以继续。
- 黄色：能运行，但有疑问。
- 红色：无法继续，需要助教。

## 7. 核心教学步骤

### 步骤 1：建立运行模型

**目标**：学生知道自己写的文本为什么能够变成输出。

在白板或投影上给出：

```text
main.py 中的代码
        ↓
Python 解释器读取并执行
        ↓
运行窗口显示结果

PyCharm：帮助我们创建、编辑、运行和调试代码的工具
```

提问：

1. 没有 PyCharm，Python 程序还能不能运行？
2. 只有 PyCharm，没有 Python 解释器，程序能不能运行？
3. `.py` 文件和 Word 文档有什么不同？

**预期结论**：PyCharm 不是 Python 本身；Python 解释器负责执行 `.py` 文件。

### 步骤 2：按环境分流

**目标**：所有学生在 10 分钟内进入一条可继续学习的路径。

按优先级分流：

```text
本地 Python + PyCharm
        ↓ 不可用
本地 Python + IDLE
        ↓ 不可用但能联网
Online Python
        ↓ 无法联网
备用电脑或同伴结对
```

主讲只询问“能否进入编辑器并运行代码”，不在此时展开安装原理。设备问题由助教单独处理。

**验证**：每位学生面前应出现一个可编辑 Python 代码的窗口。全班口令："能在自己窗口里点击光标的同学亮绿卡"，黄牌和红牌由助教立即接手。

### 步骤 3：创建项目与文件

**目标**：学生知道代码保存在哪里，也知道当前正在运行哪个文件。

PyCharm 演示顺序：

1. 选择 New Project 或新建项目。
2. 项目类型选择普通 Python 项目。
3. 项目位置使用容易找到的英文目录，例如 `python-course`。
4. 确认解释器位置显示 Python 3.x。
5. 创建文件 `main.py`。
6. 指出项目区、编辑区、运行按钮和输出区。

不要同时介绍插件、主题、Git、代码检查设置等无关功能。

**验证**：学生能够在项目区指出 `main.py`，并在编辑区点击光标。

### 步骤 4：运行传统的第一个程序

**目标**：先用最小代码完成第一次运行，不让项目背景干扰环境操作和 `print()` 的首次理解。

让学生逐字符输入，而不是复制：

```python
print("Hello, Python!")
```

运行后预期输出：

```text
Hello, Python!
```

讲解只覆盖：

- `print` 表示把内容显示到运行窗口。
- 引号之间是要输出的文字。
- 英文括号和引号必须成对出现。
- 这一行代码由 Python 解释器读取并执行。

**验证**：学生能在输出区找到完全相同的一行文字。

### 步骤 5：预测、玩梗、制造错误、再运行

**目标**：确认学生不是只完成了一次机械输入。

在 `main.py` 中加入第二行：

```python
print("Hello, Python!")
print("人生苦短，我用 Python。")
```

依次完成：

1. 运行前预测：交换第一、二行后，输出顺序如何变化？
2. 将第二行换成自己的短句，或使用“我写的不是 Bug，是隐藏功能。”
3. 新增第三行普通输出，运行并检查顺序。
4. 删除一个右括号，运行并观察红色错误信息，然后恢复。

教师此时只说明：红色信息表示 Python 没有按预期执行，它是定位问题的线索。完整错误阅读方法在下一章学习。

**验证**：学生修改代码后，输出出现相应变化；恢复括号后程序重新正常运行。

### 步骤 6：明确切换到项目集中编写

**目标**：学生已理解运行和输出后，再把本章知识集中接入贯穿项目 V0。

教师明确提示：“前面是最小语法演示，现在开始编写本章的项目版本。”新建 `v0_first_program.py`，逐行输入：

```python
print("==================================")
print("火星基地资源调度系统启动")
print("任务目标：让基地稳定运行 7 天")
print("==================================")
```

运行后预期输出：

```text
==================================
火星基地资源调度系统启动
任务目标：让基地稳定运行 7 天
==================================
```

此时只迁移已经掌握的 `print()`，不再增加新语法。让学生修改基地系统名称，并在运行前预测修改后的输出。

**验证**：学生能够说明普通演示文件 `main.py` 与项目文件 `v0_first_program.py` 分别用于什么，并成功运行 V0。

### 步骤 7：独立完成项目练习

**目标**：学生从空白文件独立完成一个最小程序。

新建 `mission_card.py`，要求输出：

```text
基地名称：学生自定
任务周期：7 天
首要目标：保持资源稳定
```

限制：至少使用三次 `print()`，不得修改已经完成的 V0 文件。

**验收**：文件名正确、能够独立运行、三行顺序正确。

## 8. 环境备用方案

| 当前情况 | 立即采用的路径 | 课后处理 |
| --- | --- | --- |
| PyCharm 正常 | 继续主路径 | 无 |
| Python 正常，PyCharm 无法启动 | 使用 IDLE | 助教课后处理 PyCharm |
| 本地环境不可用，可以联网 | 使用 Online Python | 课后完成本地安装 |
| 无法安装且无法联网 | 使用备用电脑或结对 | 记录设备和权限问题 |
| PyCharm 找不到解释器 | 助教重新选择 Python 3.x | 截图记录正确配置 |

在线环境是临时路径，不替代项目文件、解释器和本地 IDE 的学习目标。

## 9. 常见问题与排查

| 症状或错误 | 常见原因 | 处理方法 | 验证方式 |
| --- | --- | --- | --- |
| Windows 提示找不到 `python` | PATH 未配置或命令别名冲突 | 尝试 `py --version`，由助教确认安装 | 显示 `Python 3.x.x` |
| macOS 提示 `python: command not found` | macOS 通常使用 `python3` | 执行 `python3 --version` | 显示 `Python 3.x.x` |
| PyCharm 显示 No interpreter | 项目没有关联 Python | 在项目解释器设置中选择已安装的 Python 3.x | 运行按钮可用 |
| 点击运行没有当前文件 | 尚未创建运行配置或选错文件 | 右键 `main.py`，选择运行当前文件 | 输出区显示本文件结果 |
| 输出没有随修改变化 | 运行了另一个文件 | 对照运行标签和当前文件名 | 修改一行后输出同步变化 |
| `SyntaxError: invalid character '“'` | 输入了中文弯引号 | 切换英文输入法，改为 `"` | 程序可运行 |
| `SyntaxError: '(' was never closed` | 引号或括号缺失 | 从报错行向前检查成对符号，Python 3.10+ 会用 `^` 指向缺失位置 | 恢复后重新运行 |
| 文件实际为 `main.py.txt` | 系统隐藏了扩展名 | 显示文件扩展名并改为 `.py` | PyCharm 识别为 Python 文件 |
| 安装需要管理员权限 | 设备策略限制 | 不关闭安全设置，切换在线、IDLE 或备用设备 | 学生能继续完成代码任务 |

## 10. 练习与答案

### 练习 1：预测输出

```python
print("B")
print("A")
print("S")
print("E")
```

答案：按行输出 `B`、`A`、`S`、`E`。

### 练习 2：修复代码

```python
print("Hello, Python!"
```

答案：补充右括号。

```python
print(“我写的不是 Bug，是隐藏功能。”)
```

答案：将中文弯引号替换为英文半角引号。

### 练习 3：独立创作

进入项目集中练习后，学生使用三到五次 `print()` 输出一张火星任务卡。评价重点是能否独立创建、运行和修改文件，不评价故事文案长度。

## 11. 章节检查点

学生满足以下各项即可进入下一章：

- [ ] 能指出自己正在运行的 `.py` 文件。
- [ ] 能独立运行传统的 `Hello, Python!` 示例。
- [ ] 能独立输入并运行至少三行 `print()`。
- [ ] 修改一行文字后，输出能够同步变化。
- [ ] 能说出 Python 解释器和 PyCharm 的区别。
- [ ] 能在独立文件中完成项目 V0 或火星任务卡。

未通过的学生由助教使用 `mission_card.py` 再检查一次，不让其仅靠同桌代为操作。

## 12. 进度调整

### 进度快

- 增加五行任务简报排版。
- 让学生交换代码，预测对方程序输出。
- 故意制造缺少引号、括号和大小写错误。

### 进度慢

- 删除 Python 历史和应用领域介绍。
- 教师提供已创建好的 PyCharm 项目，只要求创建文件。
- 先使用 IDLE 或在线环境完成学习目标，课后再处理 PyCharm。

无论进度如何，都保留“学生独立创建并运行文件”的环节。
